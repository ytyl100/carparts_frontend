# SearchParts 管理模式功能测试指南

## 📋 新增功能概览

SearchParts 已完成管理模式的实现，包括：

1. ✅ **浏览/管理模式切换** - 仅管理员可用
2. ✅ **删除零部件** - 带确认对话框

## 🎯 功能详细说明

### 1️⃣ 浏览模式 vs 管理模式

#### 浏览模式（View Mode）
- 默认模式
- 用户只能查看信息
- 操作按钮只显示："详情"、"加入采购单"
- 详情模态框只读
- 没有删除、编辑按钮

#### 管理模式（Manage Mode）
- 需要点击 "管理模式" 按钮激活（仅管理员可见）
- 模态框和表格有橙色高亮边框（`ring-4 ring-orange-300`）
- 显示额外的管理按钮："编辑"、"删除"、"绑定分类"
- 可以直接编辑零件信息
- 可以删除零件
- 可以绑定到分类或添加新零件

### 2️⃣ 删除零部件

#### 操作步骤
```
1. 切换到管理模式
   └─ 点击搜索栏右侧的 "管理模式" 按钮

2. 点击任意行的 "详情" 按钮打开详情模态框

3. 在详情模态框中点击 "删除零件" 按钮
   └─ 弹出确认对话框

4. 在确认对话框中点击 "确认" 按钮
   └─ 零件从结果中删除

5. 关闭模态框
```

#### 确认对话框内容
```
标题: 确认删除零配件
内容: 确定要删除零件 "[零件名称]" 吗？此操作不可撤销。
按钮: [确认] [取消]
```

#### 代码实现
```typescript
const handleDeletePart = () => {
  if (!selected) return;
  setPendingAction({
    type: 'DELETE',
    title: '确认删除零配件',
    data: selected,
    action: async () => {
      setResults(results.filter(p => p.id !== selected.id));
      setSelected(null);
      setPendingAction(null);
    }
  });
};
```

### 3️⃣ 修改零部件信息

#### 可编辑字段
| 字段 | 类型 | 说明 |
|------|------|------|
| originalName | 文本输入 | 原厂名称 |
| carModel | 文本输入 | 车型 |
| brand | 文本输入 | 品牌 |
| quantity | 文本输入 | 数量 |
| origin | 文本输入 | 来源 |
| position | 文本输入 | 位置 |
| note | 文本域 | 备注 |

#### 操作步骤
```
1. 切换到管理模式

2. 点击任意行打开详情模态框

3. 在详情模态框中直接编辑字段
   └─ 每个字段都是输入框
   └─ 修改会实时更新

4. 修改状态显示
   └─ 顶部显示黄色警告条
   └─ 提示"该零件已修改，请确保保存更改"

5. 关闭模态框保存
   └─ 修改自动保存到内存状态
   └─ 下次打开会保留修改内容
```

#### 代码实现
```typescript
const handleUpdatePart = (field: keyof Part, value: any) => {
  if (!selected) return;
  const updated = { ...selected, [field]: value };
  setSelected(updated);
  setResults(results.map(p => p.id === selected.id ? updated : p));
  setModifiedParts(prev => new Set(prev).add(selected.id));
};
```

### 4️⃣ 快速选择类目并绑定

#### 场景
用户想将现有的零件绑定到特定的分类结构中，方便在分类浏览中定位该产品。

#### 操作步骤
```
1. 在详情模态框中点击 "绑定分类" 按钮
   └─ 打开类目选择器

2. 选择品牌
   ├─ 下拉菜单：[请选择品牌] [长安] [丰田] ...

3. 选择主类目
   ├─ 下拉菜单：[请选择主类目] [发动机系统] [悬挂系统] ...
   └─ 自动加载该主类目下的子类目

4. 选择子类目
   ├─ 下拉菜单：[请选择子类目] [前悬挂] [后悬挂] ...

5. 点击 "绑定分类" 按钮
   ├─ 弹出确认对话框
   └─ 确认后绑定完成

6. 关闭模态框
```

#### 确认对话框
```
标题: 确认绑定分类
内容: 确定要将此零件绑定到选定的分类吗？
按钮: [确认] [取消]
```

#### UI 展示
```
┌─────────────────────────────────┐
│ 选择分类                          │
├─────────────────────────────────┤
│ 品牌：[下拉菜单 - 选择品牌]      │
│                                 │
│ 主类目：[下拉菜单 - 选择主类目]  │
│                                 │
│ 子类目：[下拉菜单 - 选择子类目]  │
├─────────────────────────────────┤
│ [绑定分类]        [取消]         │
│                                 │
│ 或在此分类下添加新零件            │
│ [添加新零件]                      │
└─────────────────────────────────┘
```

### 5️⃣ 在指定分类下添加新零件

#### 场景
用户想在特定分类（品牌 + 主类目 + 子类目）下创建一个全新的零件记录。

#### 操作步骤
```
1. 类目选择器中完成品牌和子类目选择

2. 点击 "添加新零件" 按钮
   └─ 弹出确认对话框

3. 在确认对话框中点击 "确认"
   ├─ 新零件被添加到结果列表
   ├─ 自动设置：
   │  - id: 时间戳
   │  - subCategoryId: 选定的子类目
   │  - brand: 选定的品牌
   │  - standardName: "新零件"
   │  - position: "00000"
   │  - oeNumber: "OE-[时间戳]"
   │  - partsNumber: "PART-[时间戳]"
   │  - 其他字段: 默认值
   └─ 详情模态框打开，显示新零件

4. 可以立即编辑新零件的信息
```

#### 确认对话框
```
标题: 确认添加新零件
内容: 确定要在选定的分类下添加新零件 "新零件" 吗？
按钮: [确认] [取消]
```

## 🔗 API 接口依赖

### 当前使用的 API 方法

```typescript
// 获取零部件（已有）
MockApiClient.searchParts(term)              // 搜索零部件
MockApiClient.getPartsByCategory(category)   // 按类目获取

// 分类管理（新增）
MockApiClient.getMainCategories()            // 获取主类目
MockApiClient.getSubCategoriesByParentId()   // 获取子类目

// 零部件操作（新增）
MockApiClient.getPartById(id)                // 获取零部件详情
// (删除、修改在前端内存中实现，可选择同步到后端)
```

### 后端接口需求

```
GET /api/MainCategories              - 获取主类目列表
GET /api/SubCategories/parent/{id}   - 获取子类目列表
GET /api/CarParts/{id}               - 获取零部件详情

(可选) 实现持久化接口:
PUT /api/CarParts/{id}               - 更新零部件
DELETE /api/CarParts/{id}            - 删除零部件
POST /api/CarParts                   - 创建新零部件
```

## 💾 数据持久化说明

### 当前实现（前端内存）
- 所有修改（删除、编辑、绑定、新增）都在前端内存中进行
- 刷新页面后修改丢失
- 不调用后端删除/更新 API

### 未来改进（可选）
需要实现后端接口后，可以添加：
```typescript
// 持久化修改
await MockApiClient.updatePart(part);
await MockApiClient.deletePart(partId);
await MockApiClient.createPart(part);
```

## 🐛 故障排除

### 问题 1: 管理模式按钮不显示
**原因**: `isAdmin` 为 false  
**解决**:
- 检查用户登录状态
- 验证用户权限
- 在 App.tsx 中检查 `isAdmin` 的传递

### 问题 2: 删除后无法撤销
**预期行为**: 这是正确的，删除后操作不可撤销  
**说明**: 前端实现，后端如需持久化需另外实现

### 问题 3: 修改后立即刷新页面，修改丢失
**预期行为**: 前端内存中的修改会丢失  
**解决**: 需要实现后端持久化接口

### 问题 4: 类目选择器不显示分类
**原因**: API 请求失败  
**解决**:
- 检查后端 API 是否运行
- 查看浏览器控制台错误信息
- 验证 Token 是否有效
