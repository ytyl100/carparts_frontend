# SearchParts 管理功能快速测试清单

## 🚀 快速开始 (5分钟)

### 步骤 1: 确保条件就绪
- [ ] 后端 API 运行在 http://localhost:5017/api
- [ ] 用户已登录
- [ ] 用户具有管理员权限（isAdmin=true）

### 步骤 2: 查看管理模式按钮
- [ ] 打开 SearchParts 页面
- [ ] 在搜索栏右侧应该看到 "浏览模式" 按钮
- [ ] 如果看不到，检查管理员权限

### 步骤 3: 执行搜索
- [ ] 输入搜索关键词 (如 "8407120")
- [ ] 按 Enter 或点击搜索
- [ ] 应该显示搜索结果表格

### 步骤 4: 切换到管理模式
- [ ] 点击搜索栏右侧的模式切换按钮
- [ ] 按钮应变为橙色，显示 "管理模式"
- [ ] 表格背景应变为橙色高亮
- [ ] 每行应增加 "编辑" 和 "删除" 按钮

### 步骤 5: 测试基本功能
- [ ] 点击任意行打开详情模态框
- [ ] 验证字段变成可编辑
- [ ] 修改一个字段
- [ ] 看到黄色警告："已修改"
- [ ] 关闭模态框

## ✅ 详细测试场景

### 测试 1: 删除零部件 (2分钟)


## 🐛 常见问题排查

### Q1: 看不到模式切换按钮
```
检查清单:
□ 用户已登录?  -> 如否，请先登录
□ 用户是管理员?  -> 如否，需要分配管理员权限
□ 浏览器是否重新加载?  -> 如是，需要重新登录
□ 页面是否是 SearchParts 组件?  -> 如否，导航到正确页面
```

### Q2: 修改后刷新页面，修改丢失
```
这是预期行为，因为:
- 前端实现存储在内存中
- 页面刷新会清除内存
- 后端没有持久化接口

解决方案:
- 先关闭模态框确保修改提交
- 不要在编辑中途刷新页面
- 如需持久化，需要实现后端接口
```

### Q3: 删除按钮灰色/禁用
```
检查:
□ 是否在管理模式?
□ 是否打开了详情模态框?
□ 是否点击了正确的删除按钮?

注: 表格中的 "删除" 按钮只有在管理模式下才出现
```

### Q4: 无法选择分类
```
检查:
□ 后端 API 是否运行?
□ 分类数据是否为空?
□ Token 是否有效?
□ 浏览器控制台是否有错误?

查看错误:
- 打开 F12 开发者工具
- Console 标签页
- 查找红色错误信息
```
